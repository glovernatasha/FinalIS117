4. How the Internet Workds

Many people that use the Internet, do not understand how it works. Below, I've provided my explanation on how it works, including key terms.

In order to surf the web, you need a Domain name. A Domain Name is simply the address of the website, in which it can be accessed. For example, lets use NJIT.EDU. When you type NJIT.edu into your browser,

1. the browser connects the domain name server and receives the websites IP address

2. The browser sends for a HTTP request and checks for the entered webpage

3.Once the page is received, if found, the webpage is sent back to the browser. If the webpage is not found a 404 errors pops up(page not found).

4.Lastly, the website appears though the browser.

How to read a web address: Reading a web address is simple, and a web address(URL) is divided into the parts of:

Protocol: the request system used to retrieve the IP address for webpage

Domain: The destination

Host Name:

Domain Name: examples include: .com, .edu,.gov Port: connection for the location

Path: represents the path of the package and where ot is stored on the server

Query string: a question mark for query Fragment: represent the specifies portion of the same websites.

Lets take https://www.nytimes.com/section/technology for example:

Protocol: https://

Host Name: www.

Domain Name: nytimes

domain level: .com path: /section/technology How the internet routing works:

It is important to note that when sending information across the internet, there is no fixed path! When information is set, it is broken up into many smaller parts called packets. Each packets contains the IP address of the sender and the IP address of the receiver. Now routers come into play. A router acts like a traffic control manager and ensures that the information is receieved the fastest and information is transferred smoothly without interruptions. Another responsibility a router has is called Internet Protocol. Routers keep track of each packet and chooses the cheapest path available. So how does the internet ensures that all information we be receieved? Something called TCP (Transmission Control Protocol) comes into play. TCP basically does a check list to see if all packets are received. When TCP checks all packets, information is then displayed in its correct format. But what happens if all the packets are not received? the information will still display, but it may miss some items or not display correctly. When this happens, the webpage will resend the missing packets and go through the process again.

There are three ways a information is sent over the internet. One way being electricity. This is the most common found and can be founded in many examples such as ethernet cords. While ths option is cheaper, it is very limited in how far information can travel. Another con to having this is the increased loss of signal. Another way is light! Not only is it faster there is no signal loss. Fiber optic cables are wonderful use this use! One con of using light is that its expensive and can become hard to work with. The third way are radio waves. Radio waves are great because they are wireless but signal loss can increase from its use.

5. Linux
While creating our websites for the class, we used two distinct Linux commands. SFTP - Stands for Secure File Transfer Protocol. The successor to FTP. Works nearly the same as FTP, but with the added benefit of encrypting the files to prevent outside interference.

SSH - Stands for Secure Shell. Works in tandem with STFP by encrypting the network connection from outside threats. Most frequently used when connecting to a server that supports secure connections. We connected to NJIT AFS through a SSH and SFTp connection. From there we were able to edit our websites through webstorm.
